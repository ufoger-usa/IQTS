Alpaca Alternative Connection Guide

This guide explains how to connect to Alpaca API when the regular UI method isn't working.

Note: Only use this method if you're having issues with the regular connection interface.

Method 1: Using the alpaca-credentials.json File

The system automatically looks for a file named alpaca-credentials.json in the root directory if the regular connection fails.

Steps:

  1. Open the alpaca-credentials.json file in a text editor
  2. Replace YOUR_PAPER_KEY_ID with your Alpaca Paper Trading API key
  3. Replace YOUR_PAPER_SECRET_KEY with your Alpaca Paper Trading API secret
  4. Save the file
  5. Refresh the IQTS Trading Dashboard

Example File Format:

{
  "paper": {
    "keyId": "PK12345ABCDEF",
    "secretKey": "abcdef123456789"
  },
  "live": {
    "keyId": "AK12345ABCDEF",
    "secretKey": "xyz987654321"
  }
}
Warning: Keep your API keys secure. Don't share this file with others.

Method 2: Using the Alternative Connection Button

The system adds an "Connect Alpaca (Alternative)" button to the interface that will attempt to connect using the credentials from the JSON file.

  1. First, update the alpaca-credentials.json file as described above
  2. Look for the "Connect Alpaca (Alternative)" button in the trading panel
  3. Click the button to initiate the connection

Troubleshooting

If you're still having issues connecting to Alpaca:

  1. Check the browser console for error messages (F12 > Console)
  2. Verify your API keys are correct
  3. Ensure the alpaca-credentials.json file is in the same directory as the main HTML file
  4. Try using only the paper trading keys first before using live trading keys

For further assistance, please contact support.
